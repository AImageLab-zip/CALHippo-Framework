CLASSIFICATION README

This submodule contains the high-resolution cell classification workflow used to
assign each segmented cell polygon to one of the supported cell classes.

For the maintained public setup path, use `scripts/setup_data.py` and read:
- README.md
- documents/data_setup.md
- documents/pipeline.md

The main purpose of this stage is:
- train a classifier from manually labelled cell annotations paired with the
  original HR image crops
- apply that trained classifier to raw cell segmentations produced by the
  segmentation submodule

For the public reproducibility path, the main use is inference with the released
classifier artifacts under `data/models/classification/ml_classifier/`.


DATA NEEDED

1. Training annotations in .geojson format
   These are manually labelled cell annotations used to train the classifier.
   Each cell polygon must already contain its class label in the GeoJSON
   properties.

2. Training HR image crops in .tif format
   These are the original images paired with the training annotations.
   The current workflow expects the HR crops produced by the preprocessing
   stage, typically organized by region.

3. Segmentation outputs for inference in .geojson format
   These annotations are generated by the segmentation submodule.
   This is the main inference input of this submodule.

   For each image id there must be one .geojson file containing:
   - the ROI annotation
   - the segmented cell polygons that still need to be classified

   The classifier does not create cell polygons itself. It only assigns a class
   to the polygons coming from segmentation.

4. HR image crops for inference in .tif format
   For each segmentation GeoJSON, the corresponding HR crop must be available as
   <image_id>_HR_crop.tif.

5. Optional Hugging Face access for UNI2-h
   The `uni2h` feature backbone requires access to the gated MahmoodLab/UNI2-h
   model.

6. Released classification model artifacts
   The public reproducibility path uses released scikit-learn artifacts from
   `data/models/classification/ml_classifier/` plus the default pretrained
   `resnet18` ImageNet feature extractor.


WORKFLOW OVERVIEW

1. Prepare HR crops and annotation folders.
   The preprocessing submodule provides the HR region crops, while the training
   annotations come from manual labelling.

2. Run the segmentation submodule on the target HR images.
   This produces one GeoJSON per image containing the ROI and the segmented cell
   polygons that will be passed to classification.

3. Run `main_classification.py`.
   This script:
   - parses CLI or YAML configuration
   - selects the requested classification pipeline
   - loads the requested feature backbone if embeddings are needed
   - extracts the required cell features from the labelled training set
   - runs cross-validation for a quick training summary
   - fits the final model on all training data
   - saves the trained artifacts
   - optionally runs inference on the provided segmentation outputs

4. Review the saved model artifacts.
   The saved folder contains:
   - `model.joblib`: fitted classifier pipeline
   - `metadata.json`: resolved configuration and feature settings needed later
   - `metrics.json`: cross-validation summary and final train metrics

5. Optionally run `inference.py` later.
   This entry point loads a previously saved model folder and applies it to a
   new folder of segmentation GeoJSONs plus their corresponding HR images,
   without retraining.

Public reproducibility note:
- classification training annotations are not yet released
- the maintained public path is to download the released model artifacts and run
  `inference.py`
- using `--feature_model uni2h` requires Hugging Face access to the gated UNI2
  model


OUTPUTS

For each inference image, the submodule exports:
- `<image_id>_classification_results.geojson`
- `<image_id>_classification_visualization.png`

The GeoJSON keeps the ROI feature and writes the predicted class for each cell.
The visualization is a quick overlay of the predictions on top of the original
HR crop.
